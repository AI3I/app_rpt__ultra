## üéµ SPECIAL SLOT (00)

### CW ID Trigger (Slot 00)
| Slot | Filename | Description | CLI Command |
|------|----------|-------------|-------------|
| 00 | rpt/cw_id | **CW ID (Special)** - Triggers morse code ID using callsign and parameters from rpt.conf | `asterisk -rx "rpt playback NODENAME \|m"` |

**How It Works:**
- Slot 00 is a special trigger that plays your CW (morse code) ID
- Uses your `idtalkover` callsign from rpt.conf
- Uses pitch, speed, and other morse parameters from rpt.conf
- No .ulaw file needed - auto-generated by app_rpt
- Callable via DTMF (slot 00) or CLI command

**NEW in v2.0.5!**

## üìù NOTES

1. **Slot 00 - CW ID Trigger (NEW in v2.0.5)**
   - Special slot that plays morse code ID using rpt.conf parameters
   - No .ulaw file needed - dynamically generated by app_rpt
   - Callable via DTMF (slot 00) or CLI: `asterisk -rx "rpt playback NODENAME |m"`

2. **Slots 60-61 Weather Alerts (v2.0.5)**
   - weatheralert.sh creates temp files in /tmp/ for specific events (tornado, etc.)
   - Falls back to slots 60-61 for generic "weather alert" messages
   - You can customize these as generic fallbacks

3. **Weather Telemetry (70-79) Gets Overwritten Every 15 Minutes**
   - Editing these is pointless - cron job will overwrite them
   - They pull live data from Weather Underground API

4. **Best Practice:**
   - Use msgbuilder.sh for slots 01-59
   - Leave slots 70-79 alone (auto-generated)
   - Customize slots 60-61 if you want better generic weather alert messages
   - Slots 51-59 now available for your custom messages!
